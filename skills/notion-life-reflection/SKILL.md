# Notion 生活复盘（Life Reflection）

## 概述

为用户的 每日复盘 / 每周复盘 / 每月复盘 页面起草复盘叙述，并追加到**页面正文**（不写入字段，依用户偏好）。复盘页面已经通过 rollup 和公式聚合了指标；用户的自动流水线保持每日的关联字段连通。本 skill 把那些数字转成可读、扎根于用户真实数据的中文散文。

三个复盘数据库：

- **每日复盘** —— 有 `洞察内容`（rich_text）、`情绪`（select）、`已归档`（checkbox）、`日期`，以及 7 个 rollup（任务, 收入, 支出, 学习笔记, 专注时长, 资源）。
- **每周复盘** —— 没有手写字段；13 个 rollup + 10 个公式（周报显示, 周健康分析报告, 日复盘概览, 稳定性分析, 等）。
- **每月复盘** —— `状态`（status）、`复盘日期`、9 个 rollup + 31 个公式（综合复盘报告, 财务与消费, 健康和娱乐, 任务与管理, 知识与记录, 稳定性分析, 每周复盘详情）。

所有草稿都通过 `scripts/notion_page_io.zsh append` 写到页面正文。绝不写入字段。

**REQUIRED BACKGROUND（必备前置）：** notion-life-system —— 提供 `ntn` CLI、`scripts/notion_query.zsh`、`scripts/notion_page_io.zsh`、`references/notion-intake-rules.json`。

## 何时使用

当用户明确发起复盘时使用：

- 今日复盘 / 写日记 / 今天怎么样 / 今日洞察 / 想反思一下今天
- 周复盘 / 周报 / 本周怎么样 / 本周总结
- 月复盘 / 月报 / 这个月怎么样 / 月度总结
- 反思一下（含糊 → 反问是哪个范围）

不要用于：

- 设定目标或拆解规划 → notion-life-cascade
- 一次性记录（花钱了, 跑步了）→ notion-life-system 入库
- "本周对比上周 / 看监管数据 / 元气值如何" → notion-life-insights
- 直接编辑复盘页面字段（那些由用户的流水线管理）

## 核心循环 —— 4 步

每次复盘，按顺序执行。

1. **Locate（定位）** —— 按日期找到目标复盘页面。
   - 每日: `scripts/notion_query.zsh 每日复盘 5`，匹配 `日期` = 今天（默认）或用户指定的日期。若无对应行，新建一条并设好 `日期`；汇报后继续。
   - 每周: `scripts/notion_query.zsh 每周复盘 5`，匹配落在本 ISO 周（周一到周日）内的 `生成日期`。没有就新建。
   - 每月: `scripts/notion_query.zsh 每月复盘 5`，匹配落在本自然月内的 `复盘日期`。没有就新建。
   - 把解析到的 `page_id` 保留，供第 2、4 步用。

2. **Read State（读状态）** —— 拉取页面当前内容和渲染后的公式报告。
   - `scripts/notion_page_io.zsh get <page_id>` 返回页面的 Markdown，包含公式渲染出的单元格（进度条、"周报显示" 散文等）。
   - 每日: 提取 任务已完成 / 总数, 收入金额, 支出金额, 学习笔记个数, 专注时长, 资源总数, 以及 `情绪` 值。
   - 每周: 提取 周报显示, 周健康分析报告, 日复盘概览, 稳定性分析, 加上 13 个 rollup 数字。
   - 每月: 提取 综合复盘报告, 财务与消费, 健康和娱乐, 任务与管理, 知识与记录, 稳定性分析, 每周复盘详情。
   - 留意已有的正文内容 —— 不要重复主题。

3. **Draft（起草）** —— 用用户的第一人称口吻写平实的中文散文。
   - 起草前问用户少量问题（见下方分类小节）。可选项答案用 AskUserQuestion；开放题自由回答即可。每日: 1–3 个问题。每周: 0–1 个。每月: 0–1 个。
   - 引用具体数字但要解读。"5 个任务完成 4 个，唯一没动的是 X" 胜过 "任务完成率 80%"。
   - 平实段落，空行分隔。散文里不用 markdown 标题。不要 emoji 刷屏。
   - 篇幅指引：
     - 每日: 100–300 字（1 个亮点 + 1 个卡点 + 可选 takeaway）。
     - 每周: 300–500 字（3 个主题: 执行 / 输入 / 状态; 收尾一句下周意图）。
     - 每月: 500–1000 字（逐维度: 财务 / 任务 / 学习 / 健康 / 人际 / 内容; 收尾给下月一个优先项）。

4. **Confirm + Append（确认 + 追加）** —— 展示草稿 → 用户确认或修改 → 追加到页面正文。
   - 把最终草稿写到 `/tmp/reflection-<page_id>.md`。
   - `scripts/notion_page_io.zsh append <page_id> /tmp/reflection-<page_id>.md`。
   - 该 helper 在写入前会自动把现有正文备份到用户的 notion-page-backups 目录。
   - 汇报：page_id、备份路径、追加内容的一句话摘要。

## 三种复盘类型

### 每日

触发: 今日复盘 / 写日记 / 今天怎么样 / 今日洞察。

数据底料（rollup, 已聚合）：
- 任务已完成 / 任务总数（经 任务 关联 → 任务执行）
- 收入金额 / 支出金额（经 收入事项 / 支出事项）
- 学习笔记个数 / 专注时长（经 学习笔记）
- 资源总数（经 资源收集）
- 情绪（select —— 直接读；若为空，询问）

起草前问题（≤ 3，按需选）：
1. 今天情绪？（若 情绪 字段已设则跳过）
2. 什么让你印象最深 / 最有感觉？
3. 有什么没做完想说一下的吗？

草稿结构（≤ 3 段）：
- 整体感受 —— 锚定 情绪 + 任务完成率。
- 具体亮点或卡点 —— 锚定用户的回答 + 相关数字（学习相关用 专注时长，财务相关用 支出金额）。
- （可选）takeaway 或 明天想试试什么。

### 每周

触发: 周复盘 / 周报 / 本周怎么样 / 本周总结。

数据底料（从页面正文读，均为自动渲染）：
- 周报显示 —— 整体叙述公式
- 周健康分析报告 —— 健康专项公式
- 日复盘概览 —— 本周每日复盘聚合
- 稳定性分析 —— 方差/一致性公式
- Rollup: 任务已完成/总数, 收入, 支出, 专注时长, 学习笔记个数, 资源收集个数, 影视个数, 事项处理次数, 书籍笔记个数, 健康记录次数

起草前问题（0–1）：
- 本周想特别说一句的吗？（可选，用户说继续就跳过）

草稿结构（3 个主题，各 ≈ 100–150 字）：
- **执行** —— 任务/项目层面: 完成率 + 哪些推进 / 哪些卡住（参考 周报显示）。
- **输入** —— 学习笔记 + 专注时长 + 书籍笔记 + 影视 + 资源收集。
- **状态** —— 身体健康记录 + 事项处理 + 情绪（聚合自日复盘）+ 稳定性分析的结论。
- 收尾 1 句: 下周想保持的 / 想调整的。

### 每月

触发: 月复盘 / 月报 / 这个月怎么样 / 月度总结。

数据底料：
- 综合复盘报告 —— 顶层公式
- 逐维度公式: 财务与消费, 健康和娱乐, 任务与管理, 知识与记录
- 稳定性分析 —— 跨周方差
- 每周复盘详情 —— 4 周聚合
- 直接 rollup: 项目个数 / 已完成, 书籍已读 / 总数, 购物物品 / 金额, 美食菜谱数, 周报个数 / 归档数, 日复盘归档数

起草前问题（0–1）：
- 本月你想特别说一句的事吗？

草稿结构（~500–1000 字, 逐维度）：
- **财务** —— 收入/支出绝对值 + 跟上月对比（若 综合复盘报告 里有）+ 主要去向。
- **任务与项目** —— 项目完成率, 哪些推进 / 哪些卡住, 任务完成节奏（从 4 周稳定性看）。
- **学习** —— 书籍/笔记/专注时长 —— 知识沉淀点。
- **健康与生活** —— 饮食次数 / 健康记录 / 习惯稳定性。
- **人际与家庭** —— 事项处理频率 / 家人相关。
- **内容/创作**（若适用）—— 引用 创作计划 状态。
- 收尾 1 段: 本月最重要的一件事 + 下月最想推动的一件事。

## 通用规则

- **先 Locate 再 Draft。** 绝不在没解析出实际 page_id 的情况下往"今天"或"本月"写。
- **读现有正文。** 不要重复用户（或别的流水线）已写在本页的章节。
- **展示草稿、确认、再追加。** 三步，不抄近路。
- **追加，不替换。** 除非用户明确说 重写 / 覆盖 / 清空 —— 那时用 `notion_page_io.zsh replace`。
- **平实散文，段落分隔。** 散文里不用 `## 标题`，不用 `|表格|`。Notion 渲染简单段落很干净；markdown 花活会跟页面自身结构打架。
- **引用 + 解读数字。** 光有数字不是复盘。复盘 = 数字 + 意义。
- **少问。** 每日: ≤ 3。每周: ≤ 1。每月: ≤ 1。数据底料已经很丰富。
- **不要 emoji 刷屏。** 用户的 Notion 已经符号密集；复盘应当让人觉得平静。
- **只写页面正文。** 绝不写入 `洞察内容` 或任何其他字段 —— 依用户偏好。

## 常见错误

- 没先 `notion_page_io.zsh get` 就起草 → 重复正文里已有的内容，或跟某条自动流水线笔记矛盾。
- 罗列数字不解读 → 读起来像 CSV 倒出；用户在 Notion 里就能直接看到那些。
- 5 个以上起草前问题 → 用户失去耐心；底料就是底料，只问数据里没有的。
- 页面解析错（昨天 vs 今天，上周 vs 本周）→ 起草前永远确认你匹配到的日期。
- Notion 页面正文里用 markdown 标题 → 渲染成行内 `##` 文本，或跟已有块结构打架。
- 写入 `洞察内容` 字段而非追加到正文 → 跟用户偏好和用户的自动流水线冲突。
- 用 `replace` 而非 `append` → 抹掉自动渲染的公式 / 早先的内容。默认 `append`；只在明确 重写 时才 `replace`。

## 完整示例："今天怎么样"

1. **Locate**: `scripts/notion_query.zsh 每日复盘 5` → 找到 `日期` = 2026-05-24 的行 → page_id `22a...`。（若不存在，新建，标题 = "2026-05-24 日复盘"，`日期` = 2026-05-24。）
2. **Read State**: `scripts/notion_page_io.zsh get 22a...` → 任务已完成 = 4 / 5, 收入 = 0, 支出 = 120, 专注时长 = 85 分钟, 学习笔记 = 2, 资源 = 3, 情绪 = 平常心。现有正文有 1 条来自晨间例程的自动笔记。
3. **Ask**（情绪 已设, 跳过 Q1）: "今天印象最深的事是什么？有什么没动的吗？"
   - 用户: "下午写小红书选题写了 2 小时挺顺；早上的健身计划没去。"
4. **Draft**（~200 字, 2–3 段）: 整合 4/5 任务 + 85 分钟专注 + 选题输出 + 健身缺席 + 平常心。
5. **Confirm**: 展示草稿。用户 OK。
6. **Append**: `cat /tmp/reflection-22a.md | scripts/notion_page_io.zsh append 22a...` → 汇报 page_id + 备份路径。
